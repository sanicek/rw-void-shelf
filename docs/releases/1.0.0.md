# Void Shelf 1.0.0

Status: Accepted

## Summary

This is the first governed stable-release candidate for Void Shelf. It packages
the recovered, checksum-frozen RimWorld 1.5 payload beside the maintained
RimWorld 1.6 implementation and preserves Workshop publication ID `3008773339`.

## Changes

- Establish `1.0.0` as the canonical mod version and the GitHub repository as the
  canonical project URL.
- Preserve version-selected 1.5 and 1.6 runtime trees in one installable package.
- Keep the 1.5 Def and DLL byte-for-byte frozen under their recovered checksums.
- Build the active 1.6 assembly from maintained source and validate package
  structure, XML, metadata, routing, and frozen payload integrity.
- Establish release-candidate acceptance, deterministic archive, checksum,
  smoke-test, and publication governance.

## Update Risks

The package keeps the stable `Sanicek.VoidShelf` package ID, `VoidShelf` DefName,
and runtime class names. No save migration or external dependency is expected.
The shelf permanently destroys stored items by design; an incorrect storage
filter can irreversibly remove valuable items.

The 1.5 payload is recovered and frozen rather than rebuilt from current source.
It was not newly gameplay-tested as part of repository recovery. The active 1.6
implementation starts with an empty storage filter and Low priority, and the
exact candidate archive passed the representative gameplay test below.
